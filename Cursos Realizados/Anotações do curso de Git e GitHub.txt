Git é o maior sistema de versionamento da atualidade.

Comandos mais utilizados

git init - cria um repositório vazio ou transforma uma pasta que você já tem em um repositório.
git clone - comando para aixar o código-fonte existente de um repositório remoto como o GitHub.
git remote rm origin - desvincula a sua cópia do original
git add - adiciona os arquivos solicitados ao ambiente de stage
	usamos git add -A - para adicionar de uma vez todos os arquivos modificados.
git commit - gravação das modificações
	usamos git commit -m "descrição das atualizações do projeto"
git push origin main - salvar as modificações no repositório remoto
git pull <remote> - usado para ober atualizações do reporitório remoto
git status - permite ver quais arquivos estão sendo reastreados pelo git e quais modificações já foram enviadas para o stage.
git branch - usado para verificar todas as branches presentes no repositório
	usamos git branch -r - visualiza todas as branches
	usando git branch <branch_name> - cria uma nova branch
	devemos usar git push -u <remote> <branch_name> para upar a nova branch para o repositório remoto
